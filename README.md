## 🚀 ローカルでのセットアップ方法

### 必要な環境
- Node.js 18+
- npm 9+

### インストール & 起動

```bash
# リポジトリをクローン
git clone https://github.com/11kazu13/driver-lp-prototype.git
cd driver-lp-prototype

# 依存パッケージをインストール
npm install

# 開発サーバーを起動（http://localhost:5173）
npm run dev
```

### その他のコマンド

```bash
# プロダクションビルド
npm run build

# ビルド済みファイルのプレビュー
npm run preview

# ESLint によるコードチェック
npm run lint

# テスト実行（ウォッチモード）
npm run test

# テスト実行（CI用：1回実行）
npm run test:run
```

## 📈 CVR向上施策の概要

**Core Strategy**: 「フリクション（入力ストレス）の極小化」と「期待感・モチベーションの最大化」

### Phase 1: ファーストビューでの離脱防止

| 施策 | 内容 | 狙い |
|------|------|------|
| **お気持ち確認（Step 1）** | いきなり個人情報を聞かず「転職したい」vs「情報収集」の2択から開始 | イエスセット効果で心理的ハードルを下げる |
| **リアルタイム求人数カウンター** | "5,482件"を大きく表示 + カウントアップアニメーション | 求人が実在する期待感を醸成 |

### Phase 2: 入力フォーム最適化

| 施策 | 内容 | 狙い |
|------|------|------|
| **没入型背景デザイン** | ドライバー視点の運転席画像 + 白いカードUI | ターゲットへの親近感と視線集中 |
| **オートアドバンス** | 選択肢タップで自動ページ遷移 | 「次へ」ボタンの1アクション削減 |
| **マイクロコピー付きプログレスバー** | 「あと1分！」「折り返し地点です」など励ましメッセージ | 中だるみ防止・ゴールを意識させる |
| **郵便番号サプライズ** | 7桁入力で「あなたのエリアの平均月給は38万円です！」表示 | 住所入力への抵抗感をメリットで上書き |

### Phase 3: モチベーション維持

| 施策 | 内容 | 狙い |
|------|------|------|
| **条件連動型の求人数変動** | こだわり条件選択で上部の求人数がリアルタイム変動 | パーソナライズ感の演出 |
| **ポジティブ変換トースト** | 「その条件で歓迎の企業があります！」通知 | ネガティブ条件への引け目を払拭 |
| **バンドワゴン効果** | 「たった今、埼玉県の40代男性がエントリーしました」通知 | 社会的証明 + 適度な焦燥感 |
| **入力支援AIボタン** | Step 2〜7のみ右下に表示、CTAと被らない配置 | 迷ったユーザーの受け皿 |

### Phase 4: サンクスページ

| 施策 | 内容 | 狙い |
|------|------|------|
| **クラッカー演出 + LINE誘導** | 祝福感のある完了画面 + 「LINEで相談する」ボタン優先配置 | 電話嫌いな層の連絡到達率担保 |

## 🤖 AIの活用と自分で補った部分

### 自分で担当した部分（戦略・設計・意思決定）

| 領域 | 内容 |
|------|------|
| **施策立案** | CVR向上のためのアイデア出し|
| **改善方針** | 「何をどう改善すべきか」の判断と優先順位付け |
| **品質管理** | 実装結果のレビュー、動作確認、修正指示 |

### AIに任せた部分（実装・コーディング）

| 領域 | 内容 |
|------|------|
| **コード実装** | React コンポーネント、スタイリング、アニメーションの実装 |
| **テストコード** | Vitest + React Testing Library でのテスト作成 |
| **リファクタリング** | 日本語コメントの追加、コードの整理 |
| **トラブルシュート** | ビルドエラー、React Hooks エラーの修正 |

### この分担の意図

AIは「実装速度」と「コーディングの正確性」において人間を上回ります。
一方で、**「何を作るべきか」「なぜそれが効果的か」という戦略的判断**は人間が行うべきだと考えます。

この役割分担を明確にし：

1. **戦略・設計は自分で考え抜く** → ゴールを明確にする
2. **実装はAIに任せて爆速で進める** → 手を動かす時間を最小化
3. **アウトプットには責任を持つ** → 結果の確認と修正は必ず行う

という形で、**AIを"優秀なジュニアエンジニア"のように活用**しました。
これにより、短時間で高品質なプロトタイプを完成させることができました。

### 初期の課題分析

まずは既存のLPの構成などを洗い出した。そしてそれを元に初期仮説の既存のLPの課題を洗い出し。

**初期に感じた課題点：**

- 質問項目が画面に2個ある場合には一個ずつにフォーカスして、1つめの項目が終わったら次の項目にサクサクと進むことにすることでUXを改善する（フリクションレス）
- 該当求人数をページ上部に常にリアルタイムで表示することで求人への期待感を常に醸成し、離脱を防ぐ
- デフォルトのページはスマホ表示にすると上のヘッダー部分が文字文字しくてデザインが良くなかったのでシンプルなデザインにする
- 選択肢が二列になっているものはスマホUIだと一列にしたほうが縦にスクロールしやすい
- 郵便番号を入力した後にそれがどこなのか確認できるUIを実装する
- 生まれ年はプルダウン形式にする

### プロトタイプ作成

上記の課題点を解決しながらまた、既存のフローをきちんと網羅したプロトタイプをまずは作成した。
AIエージェント（Antigravity）にプロトタイプを作ってもらうために、まずはそのための設計書を作成し、レビューをした。
そのあと、プロトタイプを作成した。

### 完成後の改善

一旦完成したものを見ながら修正を加えていく。

1. 求人数カウンターをオレンジのカラーで動的に目立たせる
2. 文言を調整（未選択時は「ご紹介可能な求人」、選択後は「マッチする求人」）
3. カラーリングを課題で与えられたサイトに近づける
4. サンクスページに求人数は表示せず、ポジティブな要素を追加
5. プログレスバーを追加
6. 条件選択時のトースト通知を中央寄せに修正
7. 条件選択で求人数が変化する機能を追加
8. AIチャットボットボタンを追加
9. 社会的証明（バンドワゴン効果）の通知を追加
10. 郵便番号入力時に平均月給を表示
11. 最後にコードのリファクタリングとコメント付けを実施
12. テストコードを書いて品質を担保
