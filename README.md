## ローカルでのセットアップ方法

### 必要なもの
- Node.js 18以上
- npm 9以上

### インストールと起動

```bash
# リポジトリをクローン
git clone https://github.com/11kazu13/driver-lp-prototype.git
cd driver-lp-prototype

# パッケージをインストール
npm install

# 開発サーバーを起動（http://localhost:5173 で見れます）
npm run dev
```

### その他のコマンド

```bash
# 本番用にビルド
npm run build

# ビルドしたものを確認
npm run preview

# コードのチェック
npm run lint

# テストを実行（変更を監視しながら）
npm run test

# テストを1回だけ実行
npm run test:run
```

## CVR向上のために試した施策

入力のストレスを減らしつつ、「この求人サイト良さそう」と思ってもらえるような工夫を入れてみました。

### 最初に離脱されないための工夫

| やったこと | 説明 |
|------------|------|
| お気持ち確認から始める（既存LPの再現） | いきなり個人情報を聞くのではなく、「転職したい」か「情報収集」かの2択から始めることで、入力へのハードルを下げた |
| 求人数をでかく表示 | 「5,482件」みたいに求人数を目立たせて、「ちゃんと求人あるんだ」と思ってもらえるようにした |

### 入力しやすくする工夫

| やったこと | 説明 |
|------------|------|
| 背景にドライバーっぽい画像 | 運転席からの視点の画像を背景にして、「自分たちのためのサイト」感を出した |
| 選んだら自動で次へ | 選択肢を押したら勝手に次のページに行くようにして、「次へ」ボタンを押す手間を省いた |
| 進捗バーに励ましメッセージ | 「あと1分！」とか「折り返し地点です」みたいなメッセージを出して、最後まで頑張ってもらえるようにした |
| 郵便番号を入れたら平均月給を表示 | 住所を入力するのはやや抵抗あると思うので、「この地域の平均月給は38万円です」みたいな情報を出してメリットを感じてもらうようにした |

### やる気を維持してもらう工夫

| やったこと | 説明 |
|------------|------|
| 条件選択で求人数が変わる | こだわり条件を選ぶたびに上の求人数が変わるようにして、「自分の条件で検索してくれてる感」を出した |
| 「その条件で歓迎の企業があります」通知 | ネガティブな条件を選んでも「大丈夫ですよ」と安心させるメッセージを表示 |
| 「誰かがエントリーしました」通知 | 「たった今、埼玉県の40代男性がエントリー」みたいな通知を出して、「他の人も使っているんだ」という感じを出した |
| AIに相談ボタン | 入力中に困ったらAIに相談できるボタンを右下に配置（今はダミーです） |

### 完了後

| やったこと | 説明 |
|------------|------|
| お祝い感のある完了画面 | ただの「完了しました」じゃなくて、達成感のあるデザインにした |
| LINEで相談ボタン | 電話が苦手な人向けにLINEボタンを目立たせた |

## アピールしたい工夫

### 1. スピード感

課題をもらってから約3時間くらいで一通り作りました。
もちろんAIをガンガン使ったのでスピードが出せたというのはあります。

やったこと：
- 7ステップのフォーム
- 10個以上のCVR向上施策
- スマホ向けのUI
- コードへの日本語コメント追加
- テストコード20ケース

### 2. 学習期間の割には頑張った（と思う）

プログラミングを始めてまだ3ヶ月くらいですが、AIを上手く使えばこれくらいのものは作れるようになりました。
正直、まだまだ分からないことだらけですが、分からないことをAIに聞きながら進められるのは強みかなと思っています。

### 3. とりあえず動くものを作る

完璧を目指すより、まず動くものを作って、それを見ながら改善していく方が早いと思っているので、そういう進め方をしました。
実務でも同じように、スピード感を持ってアウトプットを出していきたいです。

## AIの活用と自分で補った部分

### 自分でやったこと

| 何を | 具体的に |
|------|----------|
| アイデア出し | 「こうすればCVR上がりそう」という施策を考えた |
| 方針決め | 何をどの順番でやるか決めた |
| 確認と修正 | AIが作ったコードが動くか確認して、おかしいところは直した |

### AIにやってもらったこと

| 何を | 具体的に |
|------|----------|
| コード書き | Reactのコンポーネントとか、CSSとか、アニメーションとか |
| テスト作成 | テストコードを書いてもらった |
| コメント追加 | 日本語でコードの説明を書いてもらった |
| エラー修正 | ビルドエラーとかが出たときに直してもらった |

### なぜこの分担にしたか

AIはコードを書くのが速くて正確なので、そこは任せました。
でも、「何を作るか」「なんでそれが良いのか」は自分で考えないといけないと思ったので、そこは頑張りました。

やり方としては：
1. 自分で「こういうのを作りたい」とゴールを決める
2. AIに実装してもらう
3. 動くか確認して、ダメなら修正してもらう

AIに丸投げするんじゃなくて、ちゃんと自分で結果を確認して、責任を持つことは意識しました。

### 最初に考えた課題

既存のLPを見て、気になった点：

- 質問が2個同時に出てると入力しづらい → 1個ずつ出す
- 求人数がよく分からない → 上に常に表示する
- スマホで見るとヘッダーがごちゃごちゃ → シンプルにする
- 選択肢が2列だとスマホで押しにくい → 1列にする
- 郵便番号入れても住所が見えない → 表示する
- 生年月日が入力しづらい → プルダウンにする

### 作った流れ

1. 課題を整理して、AIに設計書を作ってもらった
2. 設計書をレビューしてOKだったらプロトタイプを作成
3. 動かしながら気になったところを修正

### 修正した内容

1. 求人数カウンターを目立たせた
2. 文言を調整した
3. カラーを元のサイトに近づけた
4. サンクスページを改善した
5. プログレスバーを追加した
6. トースト通知の位置を修正した
7. 条件選択で求人数が変わるようにした
8. AIボタンを追加した
9. 「誰かがエントリーしました」通知を追加した
10. 郵便番号入力時に月給を表示するようにした
11. コードを整理してコメントを追加した
12. テストを書いた
