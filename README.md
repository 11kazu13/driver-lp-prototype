質問項目が画面に2個ある場合には一個ずつにフォーカスしていっこめの項目が終わったら次の項目にサクサクと進むことにすることでuxを改善する（フリクションレス）
該当求人数をページ上部に常にリアルタイムで表示することで求人への期待感を常に醸成し、離脱を防ぐ
デフォルトのページはスマホ表示にすると上のヘッダー部分が文字文字しくてデザインが良くなかったのでシンプルなデザインにする

選択肢が二列になっているものはスマホuiだと一列にしたほうが縦にスクロールしやすいと思うのでそれを反映する

郵便番号を入力した後にそれがどこなのか確認できるuiを実装する

生まれ年はプルダウン形式にする

上記の課題点を解決しながらまた、既存のフローをきちんと網羅したプロトタイプをまずは作成した

その後感じたLP改善のためのものや修正したところ↓

まず一番初めのページで該当求人数のところをオレンジのカラーとかで動的に目立たせるように囲むようなデザインをしたい
目的は求人が存在することの期待感を煽るためです　この変更を実施した

上記の結果見た目はいい感じになったのだが、まだ何も選択とか情報を与えていないタイミングであなたにマッチするとか言われてもあまりピンとこないと思うから
文言を調整した

カラーリングは課題で与えれたサイトに近いほうがいいなと感じたのでそれを修正

サンクスページに、求人の数とかは表示しなくていいかなと思ったのでそれを修正する
その代わりにおめでとうみたいなポジティブな要素を入れたり、
そのまま既存の求人を確認する、みたいなボタンを表示させる感じにしたい（実際に遷移するところは実装しなくてもいいです。）
もしくは、ラインで相談できるためのボタンも追加してもいいかも

全てのページの上部に今全体のフローの中のどこにいて後どれくらいで終わるのかみたいなステップ数が視覚的に見えるようになる機能を実装→これにより離脱を防ぐ（プログレスバーっていうのかな？

その条件で歓迎の企業があります！→これを入れることにより、条件に合う企業があるという期待感を高める
というpopがq6のところに出てくるんだけど
それが左右の中で右側によってしまっているから高さは変えなくてもいいので中心に表示されるようにする

ここでそれぞれの条件を選択した際にも上部の該当求人数が変化するようにしたい。
数値は一旦適当でいいのでフロント部分の表示だけ調整した

「誰かがエントリーしました」通知（バンドワゴン効果）
画面の隅（例えば左下）に、邪魔にならない程度で**「たった今、埼玉県の40代男性がエントリーしました」**といった小さな通知を数秒おきに出す。
効果: 「みんな使っているんだ」という安心感（社会的証明）と、「自分も早くしなきゃ」という焦燥感を煽ります。
